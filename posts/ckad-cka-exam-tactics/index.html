CKAD / CKA exam tactics

Tags

cka ckad k8s kubernetes

The CKAD (Certified Kubernetes Application Developer) and CKA (Certified Kubernetes Administrator) certifications probably need no introduction to anyone who is a k8s user.

In mid May 2020, I obtained both certifications and wish to share some pointers with you.

Unlike many other resources similar to this post that cover the curriculum, this blog post takes a slight twist in emphasizing some concrete non k8s related preparation work I did upfront to sharpen my edge for the exam, as well as concrete exam tactics I used during the exam.

Ground rules

There are some rules enforced by CNCF and must be strictly adhered to during the exam. These are the ones I find very important:

  • The room must be well lit. However, there should be no bright lights or window directly behind you.
  • Clear desk policy - other than what is required (computer, keyboard, mouse, monitor, webcam), nothing else can be on the desk. (I asked if I could leave my id on the table after showing it to the proctor and he/she told me to keep it)
  • You may have water in a clear glass or unmarked water bottle. This is the only exception to the above.
  • In your Chrome web browser, only 2 tabs may be open: one for the exam, and another for documentation to allowed websites (https://kubernetes.io/docs, https://kubernetes.io/blog, https://github.com/kubernetes, and any subdirectories under those sites)
  • A moveable webcam is required. Before the exam starts, the proctor (examiner) will request you to move your webcam to scan the entire room and under your desk. I used a Macbook and its built-in webcam so any laptop with a built-in webcam will suffice.
  • For the duration of the exam, no one else can enter the room.

Once the exam starts:

  • You may not move away from the computer without permission of the proctor. Breaks may be requested. Each break may be up to 15 minutes.
  • You may not read out any part of the question. Not even silently / “mouthing it”.
  • You may not cover any part of your face for extended periods of time.

Prerequisite skills

In addition to the required k8s knowledge, you should have a good grasp of the following before you attempt the exam:

  • vim (preferrably; otherwise emacs or nano or a similar Linux command line text editor)
  • some knowledge of the Linux command line
  • screen / tmux (optional but helpful because you only have 1 terminal in the exam)

The exam requires you to use the Chrome web browser and the terminal exposed to you is that of a Linux machine. As such, you have to know the above to pass the exam.

Preparation work I did (non k8s practice related)

These are what I did and recommend candidates to do:

  • Ensure your system fulfils the requirements. You will need to install a Chrome extension which will let you test your system for it. Once you have purchased the exam at the Linux Foundation Portal, there is a link for you to go to https://www.examslocal.com/ScheduleExam/Home/CompatibilityCheck to do the compatibility check.
  • Backup irrelevant browser bookmarks and delete them. If possible, delete irrelevant browser extensions as well. This is to decrease the probability of you accidentally accessing websites not permitted by the exam rules.
  • Bookmark useful documentation in https://kubernetes.io/docs that are useful. To find out more, read my CKAD and/or CKA specific posts.
  • Figure out how to copy and paste from/to the web interface - different OS require different commands. The CKAD / CKA guide should provide this (for Mac, it’s the usual CMD-c and CMD-v)
  • Do some practice on the actual computer you will be taking the exam on to get used to it. For myself, I used a Macbook to to take the exam because it has a webcam, as well as an external keyboard - both pieces of hardware are not what I use normally.
  • If your desk is cluttered, spend some time the day before to clear everything irrelevant from it.
  • Memorize vimrc if using vim. More on that later.
  • Memorize bash aliases for kubectl commands so you save on typing during the exam. More on that later.

Memorize vimrc

If you are a heavy vim user (like myself) and/or will be using vim during the exam, I highly recommend you to memorize a bare bones vimrc and input it during the start of the exam so you get some good defaults. I used the following for the exam:

set nocompatible
set et
set number
set ai
set ts=2
set sw=2
set sts=2
set bs=2

Memorize bash aliases for kubectl commands

bash aliases for commonly used kubectl commands will save you a ton of typing (and time). I used the following for the exam:

alias v=vim
alias k=kubectl
alias kg='k get'
alias kdes='k describe'
alias kd='k delete'
alias kdf='kd -f'
alias kaf='k apply -f'
alias kx='k explain'
alias kcc='k config current-context'
alias kn='k config set-context $(kcc) --namespace'
alias knd='kn default'
alias knks='kn kube-system'
alias kcu='k config use-context'
alias kl='k logs'
alias klf='kl -f'

You should come up with your own list.

During the exam, you can append this line to ~/.bashrc to source for the aliases:

[ -f ~/aliases ] && source ~/aliases

One day before the exam

Please have a good rest. If you have done your preparation work, you should have no problem passing the exam. For CKAD, you should be aiming for a perfect score. For CKA, the content is slightly harder but you should be aiming for a very high score.

On the day of the exam itself

Access the exam site 15 minutes prior to the scheduled time. You should do so by logging onto the Linux Foundation Portal. There will be a link to redirect you to the examslocal.com site.

The proctor will come in, do the ID check and lay down ground rules before releasing exam to you.

Once the exam was released to me, I did the following:

  • Ran vim ~/.vimrc, entered the vimrc contents above, saved and quit.
  • Ran vim ~/aliases, entered the bash aliases, modified ~/.bashrc to source those aliases.
  • Create generic PodSpec: k run nginx --restart=Never --image nginx --dry-run -o yaml >podspec.yml
  • Create generic Deployment Spec: k create deployment nginx --image nginx --dry-run -o yaml >deployspec.yml

The generic PodSpec and Deployment Spec come in handy when new Pods / Deployments need to be created but their complexity overwhelms the kubectl run or kubectl create command line. Simply make a new copy, make required modifications on that copy, then kubectl apply -f.

Exam tactics

  • The biggest constraint in both exams is time, especially the CKAD exam. Questions have points associated and can be done independently of one another. Therefore, like many others who have written similar guides, I highly recommend starting from the questions with more points.
    • Such questions are not necessarily more difficult than questions with less points. A useful tactic I picked up from one exam guide and used during the CKA exam is to use the in-built notepad to make a list of all the questions and their points, then start from the question with the most points that I am confident of doing.
  • (IMPT) There are multiple k8s clusters. Before rushing into a question, remember to copy and paste the kubectl command supplied by the question to change to the correct context. Otherwise you would have done your work for nothing.
  • Ensure you are in the correct namespace. This is as important as the previous point. If no namespace is stated in the question, you are to use the default namespace.
    • If using manifest files, for resources that are scoped to a namespace, you should explicitly specify the namespace in the metadata section.
    • If using the command line, ensure you have switched to the correct namespace; I recommend resetting the namespace to default right after you have issued the command to switch context. If you are using my bash aliases, running knd will do the job.
  • If you are stuck, move on. The web interface contains a button for you to flag a question so you can return to it later. Flagged questions will have a blue flag next to them in the dropdown menu used for question selection. You may also use the text editor in the web interface to record down the information, or you can just use vim / nano in the terminal.
  • The web interface is built so that it is easy to copy and paste information (which you may type incorrectly under stress) such as Pod name, Deployment name, container name, labels, etc using a single click. No excuses for typos!
  • Always be on the “main node” unless you need to do work in another node. I use quotes here because I forgot the name of the actual main node; you will find out what it is during the exam.

On the web interface

I have read some posts that suggest the web interface is crappy. My personal experience is otherwise; I find it pretty decent.

In terms of layout, there is a control section at the top of the page. The questions are on the left hand side of the page. The terminal is on the right hand side of the page and takes up the majority of the space.

Control section

From what I can remember, there are buttons / dropdown menus to:

  • Share your camera and microphone with the proctor.
  • Share your screen with the proctor.
  • Open up a live chat with the proctor (if you have closed it).
  • Open up a text editor (contents are preserved when you close it).
  • Request a break.
  • Request ending the exam.
  • Safely refresh the page.

Questions section

The most important thing about this section is that important information such as namespace, Pod names, Deployment names and the like can all be copied. Hence you should not be making any typos.

There is a button at the end of every question to flag it. A dropdown menu is available to quickly navigate through questions. Questions that are flagged will have a blue flag appear next to them for quick identification.

Terminal section

There are 2 parts to this that can be toggled via “tabs”. The first is a help page that shows a set of instructions available using the man lf_exam command. The next is the terminal itself.

Because there is only 1 terminal, if you foresee yourself requiring multiple terminals, use a terminal multiplexer such as screen or tmux.

If you lose your connection

This happened to me during my CKAD exam. About an hour into the exam, I lost my connection to the exam. My Internet connection was alright but somehow the exam website said I lost my connection to the exam and showed instructions to either call the support number or email the given support email address. None of that worked for me.

If this happens to you, please do not panic. When I took the CKA exam, I asked my proctor about this and what he / she suggested was to just wait because it is most likely an issue on the proctor’s end. The timer will be paused until you can resume the exam.
